Impact Repository

Impact source codes are kept in Sourceforge CVS (Concurrent Versions System) repository.

CVS (Concurrent Versions System) is a tool used by many software developers to manage changes within their source code tree. CVS provides the means to store not only the current version of a piece of source code, but a record of all changes (and who made those changes) that have occurred to that source code. Use of CVS is particularly common on projects with multiple developers, since CVS ensures changes made by one developer are not accidentally removed when another developer posts their changes to the source tree.

Though CVS repositories are most commonly accessed using a special piece of software called a CVS client, Sourceforge also provide a web-based interface to view CVS repositories. Browsing the CVS tree gives you a great view into the current status of this project's code. You may also view the complete history of any file in the repository.

Using CVS Client

The most convenient way to access Impact CVS repository and to follow any updates by the developers and to commit changes (if you are the developer and have write access to the repository) is by using CVS client.

There are many free CVS clients available. Eclipse is one of them that supports Windows, Linux and Mac operating systems. In Windows OS, you need to download first then install it to your computer. For Ubuntu users, just select Eclipse from Ubuntu Software Center and click Install.

Anonymous CVS Access

Impact in SourceForge CVS repository can be checked out through anonymous (pserver) CVS. Password is not required but no write access is given. Follow the steps below in Eclipse:

  1. Select a new project directory
  2. Create a new project (File > New > Project)
  3. Select CVS > Projects from CVS
  4. Enter repository location information
    Host: impact.cvs.sourceforge.net
    Repository path: /cvsroot/impact
    Username: anonymous
  5. Select impact module and click Finish button. The source codes will be downloaded to your local folder defined in step 1.

If you have modified the codes to improve Impact or you have new codes to implement new features, please send them to Impact project administrator to review. You might be granted a developer access to the repository after reviewing the codes.
